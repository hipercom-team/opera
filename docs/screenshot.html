In this page, we present some snapshots of OPERA running on sensors.

 OCARI workshop participants at the demonstration

It is in the occasion of an open demonstration of the OCARI project in December 2011 that we have demonstrated the functioning of OPERA on Telit hardware.

 This is a CC2420 sensor mote with 8 Kbytes of RAM

OPERA was also ported to this hardware - a TelosB mote.

 This is a wireless sensor testbed

Based on this wireless sensor testbed where sensors form a grid, we visualize hereafter some results using a tool developed in the team. This tool is able to monitor the network, sniff control packets and show the running protocols (messages exchanged, topology, colors, etc).

 Topology Construction

This image shows the exchange of Hello messages that enable sensors to detect symmetric links (double arrows). Node 16 is the data sink.

 Routes Construction

This image shows the exchange of STC message used by EOLSR to build routes from any node to the STC generator. In this example, it is the node 16 that generates this message to build the tree. We see on the right side the built tree rooted at node 16.

 Topology Coloring

This is a wireless sensor testbed.

This image shows the exchange of colored topology performed by OSERENA via the messages Color. We observe that nodes 14 and 20 that are 4 hops away share the same color.
